# Epic 1 Training Dataset Structure for Claude Generation

## Overview
This document defines the comprehensive data structure for training data that will be generated by Claude to improve the Epic 1 multi-view ML query complexity analyzer.

## Core Data Structure

### TrainingDataPoint
```python
@dataclass
class TrainingDataPoint:
    """Single training example with all view scores and metadata."""
    
    # Core data
    query_text: str
    expected_complexity_score: float  # 0.0-1.0 final complexity
    expected_complexity_level: str    # "simple", "medium", "complex"
    
    # Individual view scores (ground truth)
    view_scores: Dict[str, ViewScore]
    
    # Metadata for quality assessment
    metadata: TrainingMetadata
    
    # Generated features (optional - for validation)
    extracted_features: Optional[Dict[str, Any]] = None

@dataclass
class ViewScore:
    """Individual view assessment with detailed breakdown."""
    
    view_name: str                    # "technical", "linguistic", etc.
    complexity_score: float          # 0.0-1.0 score for this view
    confidence: float                # 0.0-1.0 confidence in assessment
    
    # Key contributing factors
    primary_indicators: List[str]    # Main complexity indicators found
    feature_values: Dict[str, float] # Specific measurable features
    reasoning: str                   # Human-readable explanation
    
    # Statistical properties
    expected_distribution: str       # "normal", "bimodal", "uniform"
    difficulty_factors: List[str]    # Specific challenge aspects

@dataclass
class TrainingMetadata:
    """Metadata for quality assessment and dataset management."""
    
    # Generation metadata
    generation_timestamp: datetime
    claude_model: str                # Model used for generation
    prompt_version: str             # Version of generation prompt
    
    # Query characteristics
    domain: str                     # "technical", "general", "academic"
    query_type: str                 # "how-to", "definition", "troubleshooting"
    complexity_category: str        # Primary complexity driver
    
    # Quality indicators
    quality_score: float            # 0.0-1.0 overall quality assessment
    validation_flags: List[str]     # Quality validation results
    human_review_needed: bool       # Requires human validation
    
    # Dataset balancing
    target_complexity_level: str    # Intended complexity for balancing
    difficulty_subcategory: str     # Fine-grained difficulty classification
```

## Complete Dataset Structure

### TrainingDataset
```python
@dataclass
class TrainingDataset:
    """Complete training dataset with metadata and validation."""
    
    # Core data
    datapoints: List[TrainingDataPoint]
    
    # Dataset metadata
    metadata: DatasetMetadata
    
    # Statistical validation
    statistics: DatasetStatistics
    
    # Quality assessments
    quality_metrics: QualityMetrics

@dataclass
class DatasetMetadata:
    """High-level dataset information."""
    
    name: str
    version: str
    creation_date: datetime
    total_samples: int
    
    # Generation parameters
    generation_strategy: str        # "balanced", "targeted", "comprehensive"
    claude_prompts_used: List[str]  # Prompt versions used
    
    # Dataset composition
    complexity_distribution: Dict[str, int]  # Count per complexity level
    domain_distribution: Dict[str, int]      # Count per domain
    query_type_distribution: Dict[str, int]  # Count per query type
    
    # Quality targets
    target_accuracy: float          # Desired model accuracy
    minimum_samples_per_class: int  # Balancing requirement
```

## Statistical Validation Framework

### Quality Metrics Structure
```python
@dataclass
class QualityMetrics:
    """Comprehensive quality assessment for the dataset."""
    
    # Individual datapoint quality
    individual_scores: List[float]  # Quality score per datapoint
    outlier_indices: List[int]      # Datapoints requiring review
    
    # Inter-view consistency
    view_correlation_matrix: Dict[str, Dict[str, float]]
    consistency_scores: List[float] # Per-datapoint consistency
    
    # Distribution quality
    complexity_balance_score: float # How well-balanced across levels
    domain_coverage_score: float    # Coverage of different domains
    feature_distribution_health: Dict[str, float]  # Per-feature distribution quality
    
    # Validation results
    statistical_tests: Dict[str, Any]  # Formal statistical validation
    quality_flags: List[str]           # Issues found during validation
    recommendations: List[str]         # Improvement suggestions

@dataclass
class DatasetStatistics:
    """Statistical summary of the dataset."""
    
    # Basic statistics
    total_samples: int
    samples_per_complexity: Dict[str, int]
    
    # Score distributions
    complexity_score_stats: Dict[str, float]  # mean, std, min, max
    view_score_distributions: Dict[str, Dict[str, float]]
    
    # Feature statistics
    feature_correlations: Dict[str, Dict[str, float]]
    feature_importance_scores: Dict[str, float]
    
    # Quality indicators
    label_consistency_rate: float    # Agreement between views
    coverage_completeness: float     # Feature space coverage
    sample_diversity_score: float    # Diversity of examples
```

### Quality Validation Example
```python
def validate_single_datapoint(datapoint: TrainingDataPoint) -> List[str]:
    """Validate individual datapoint quality."""
    issues = []
    
    # Check view score consistency
    view_scores = [v.complexity_score for v in datapoint.view_scores.values()]
    if np.std(view_scores) > 0.4:  # High disagreement
        issues.append("high_view_disagreement")
    
    # Check feature-score alignment
    if not validate_feature_consistency(datapoint):
        issues.append("feature_score_mismatch")
    
    # Check expected vs actual complexity
    if abs(datapoint.expected_complexity_score - np.mean(view_scores)) > 0.3:
        issues.append("complexity_inconsistency")
    
    return issues
```
